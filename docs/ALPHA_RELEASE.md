# Alpha Release Playbook

This guide is for producing Congruity alpha builds via Tauri for:
- macOS
- Windows
- Android
- iOS

All commands assume repo root at:
- `/Users/coltonspurgin/Developer/congruity`

## 1) Pre-flight

### Required env
Set these for the client build:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_SIGNALING_URL`

### Quality gate (required before artifacts)
```bash
cd client
npm run lint
npm run test:run
npm run test:coverage
```

Coverage thresholds are enforced in Vitest config.

## 2) Desktop builds

### macOS (run on macOS host)
```bash
cd client
npm run tauri:build:macos
```

### Windows (run on Windows host or CI windows runner)
```bash
cd client
npm run tauri:build:windows
```

## 3) Mobile setup (one-time)

### Android init
```bash
cd client
npm run tauri:mobile:init:android
```

### iOS init
```bash
cd client
npm run tauri:mobile:init:ios
```

## 4) Mobile builds

### Android
```bash
cd client
npm run tauri:android:build
```

### iOS (run on macOS + Xcode)
```bash
cd client
npm run tauri:ios:build
```

## 5) Toolchain requirements

### Desktop
- Rust toolchain
- Node.js + npm
- Tauri CLI (already in `devDependencies`)

### Android
- Android Studio / Android SDK
- Java (JDK 17+ recommended)
- `ANDROID_HOME` and platform-tools in `PATH`

### iOS
- macOS host
- Xcode + command line tools
- valid Apple signing identity / provisioning profile for device/TestFlight distribution

## 6) Artifact locations

Tauri desktop artifacts are emitted under:
- `client/src-tauri/target/release/bundle/...`

Android/iOS artifacts are generated by their native build systems after Tauri mobile build commands complete.

## 7) Notes for alpha

- Use HTTPS/WSS signaling endpoints for internet-facing usage.
- Keep `signaling` and `minio` behind Cloudflare tunnel or equivalent TLS edge.
- For iOS/Android external distribution, add signing and store packaging steps in a follow-up release workflow.
